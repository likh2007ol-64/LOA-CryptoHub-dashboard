Создай новый Python-проект с названием "loa-crypto-dashboard" для Streamlit веб-дашборда платформы LOA-CryptoHub.

Требования к дашборду:

1. Технологии: Python 3.10+, Streamlit, requests, pandas, plotly (для графиков).

2. Функциональность (согласно ТЗ, уровень доступа ЭКСПЕРТ и администратор):
   - Главная страница: приветствие, общая статистика (курсы топ-10 криптовалют в реальном времени через API).
   - Страница "Портфель": добавление/удаление монет, расчёт P&L, график стоимости портфеля.
   - Страница "История": транзакции портфеля.
   - Страница "Уведомления": список ценовых оповещений, создание/удаление.
   - Страница "Отчёты": подписка на ежедневные отчёты.
   - Страница "Безопасность": статус сессии, кнопка "Удалить мои данные".
   - Страница "Админ-панель" (только для администратора): управление пользователями, выдача подписок, просмотр логов.
   - Страница "Мониторинг и диагностика" (только для администратора): статус компонентов (VK-бот, API, БД, внешние API), цветовая индикация, таблица проверок, кнопки управления.

3. Интеграция с API:
   - Базовый URL API должен читаться из переменной окружения API_BASE_URL (по умолчанию http://localhost:8080).
   - Использовать эндпоинты: /api/v1/prices, /api/v1/user/{user_id}, /api/v1/portfolio, /api/v1/alerts и т.д.
   - При отсутствии API показывать заглушку.

4. Авторизация:
   - Вход через VK ID (имитация: ввод user_id в sidebar, затем загрузка данных пользователя через API).
   - Для MVP достаточно простого ввода user_id, но с проверкой, что такой пользователь существует.

5. Темы оформления:
   - Поддержка тёмной, светлой, неон, синей тем (выбирается через sidebar).
   - Тёмная тема по умолчанию (фон #0A2540, карточки #1E1E2F, акценты #00F0A0 и #F5B041).

6. Структура файлов:
   loa-crypto-dashboard/
   ├── app.py                 # главный файл Streamlit
   ├── pages/                 # многостраничный дашборд
   │   ├── 1_Портфель.py
   │   ├── 2_История.py
   │   ├── 3_Уведомления.py
   │   ├── 4_Отчёты.py
   │   ├── 5_Безопасность.py
   │   ├── 6_Админ_панель.py
   │   └── 7_Мониторинг.py
   ├── utils/
   │   ├── api_client.py      # функции запросов к API
   │   └── theme_manager.py   # управление темами
   ├── requirements.txt
   └── .streamlit/
       └── config.toml        # настройки темы и порта

7. Запуск:
   - Команда: streamlit run app.py --server.port 8501
   - Порт 8501 должен быть открыт.

8. Дополнительно:
   - Скопируй файлы API-клиента из проекта loa-crypto-api (если доступны) или создай новые.
   - Используй мок-данные, если API недоступно, но в продакшене должно работать с реальным API.
   - После создания файлов установи зависимости (pip install -r requirements.txt) и запусти дашборд.
   - Покажи мне URL, где дашборд доступен (внутри Replit будет вкладка "Web Preview").
