Ты находишься в проекте `Crypto Dashboard` (Streamlit дашборд). Выполни доработку для интеграции с реальным API.

1. **Конфигурация API**:
   - В файле `.env` (или в коде) задай `API_BASE_URL = "http://153.80.184.34"` (адрес API на VPS).
   - Убери все мок-данные, все вызовы должны идти к API.

2. **Авторизация**:
   - Оставь ручной ввод VK ID (поле ввода) – для тестирования. При вводе ID дашборд должен запрашивать через API `GET /api/v1/users/{vk_id}` и получать информацию о пользователе (план, роль администратора и т.д.).
   - Для администратора (роль admin или входит в число ADMIN_IDS) показывать страницы "Админ-панель" и "Мониторинг".

3. **Страницы дашборда** (согласно ТЗ):
   - **Главная**: общая статистика (курсы в реальном времени, график топ монет).
   - **Портфель**: добавить/удалить монеты, расчет P&L, график стоимости. Данные сохранять через API (`POST /api/v1/users/{vk_id}/portfolio`, `GET /api/v1/users/{vk_id}/portfolio`, `DELETE /api/v1/users/{vk_id}/portfolio/{symbol}`).
   - **История**: транзакции портфеля (`GET /api/v1/users/{vk_id}/transactions`).
   - **Уведомления**: создание/удаление ценовых алертов (`GET/POST/DELETE /api/v1/users/{vk_id}/alerts`).
   - **Отчёты**: подписка на ежедневные отчёты (`POST /api/v1/reports`, `GET /api/v1/users/{vk_id}/reports`).
   - **Безопасность**: кнопка "Удалить мои данные" (`DELETE /api/v1/admin/users/{vk_id}/data` для администратора, а для обычного пользователя – удаление своих данных (эндпоинт нужно создать самостоятельно, либо использовать тот же, но с проверкой прав). Пока можно реализовать как заглушку.
   - **Админ-панель** (только для admin): просмотр пользователей, выдача подписок (`POST /api/v1/admin/users/{vk_id}/subscription`), просмотр логов (`GET /api/v1/admin/logs`), удаление данных пользователей.
   - **Мониторинг** (только для admin): отображение статуса компонентов (можно использовать `GET /health` для API и проверять доступность бота через отдельный эндпоинт, или пока сделать заглушку).

4. **Технические требования**:
   - Все запросы к API делать через `requests` (синхронно) или `aiohttp` (асинхронно, но Streamlit синхронный). Используй `requests`.
   - Обрабатывай ошибки: если API недоступен, показывай предупреждение, но не падай.
   - Используй `st.cache_data` для кэширования курсов (на 30 секунд).

5. После внесения изменений выведи сообщение: "✅ Дашборд интегрирован с реальным API. Готов к тестированию."

Не удаляй существующие страницы, а дополняй их реальными вызовами API. Если каких-то эндпоинтов не хватает в API, создай заглушки (выводи сообщение "функция в разработке").
