Ты находишься в проекте Crypto Dashboard (Streamlit). API уже полностью готово и доступно по адресу, указанному в переменной окружения API_BASE_URL (например, https://api.loa-crypto.ru или http://153.80.184.34). Твоя задача — доработать дашборд, чтобы он использовал все возможности API и соответствовал ТЗ.

### Что нужно сделать

1. **Обновить `utils/api_client.py`**:
   - Добавить функции для всех новых эндпоинтов API:
     - Управление кошельками: `get_wallets`, `add_wallet`, `delete_wallet`
     - AI-сигналы: `get_signal`, `get_all_signals`
     - DeFi-аналитика: `get_tvl`, `get_apy`, `get_gas_price`
     - Админ-мониторинг: `get_health_all`, `get_metrics`, `get_system_info`
     - Экспорт CSV: `export_portfolio_csv`, `export_transactions_csv`, `export_alerts_csv`, `export_reports_csv`
   - Использовать асинхронный `aiohttp` (или синхронный `requests`, как в остальном коде). Для совместимости с Streamlit можно использовать `requests` с `@st.cache_data`.
   - Обрабатывать ошибки и возвращать заглушки при недоступности API.

2. **Создать новые страницы дашборда (файлы в папке `pages/`)**:

   - **Страница «Кошельки» (`6_Кошельки.py`)**:
     - Просмотр списка кошельков пользователя.
     - Добавление нового кошелька (адрес, блокчейн, метка).
     - Удаление кошелька.
     - Отображение баланса и истории изменений (если API предоставляет).
   - **Страница «AI-сигналы» (`7_AI_сигналы.py`)**:
     - Таблица всех монет с прогнозом (бычий/медвежий/нейтральный), вероятностью, RSI, MACD.
     - Детальный вид для каждой монеты.
   - **Страница «DeFi» (`8_DeFi.py`)**:
     - Общий TVL, топ-10 протоколов.
     - Поиск APY по символу (например, USDT).
     - Текущая цена газа (Ethereum, BSC и т.д.).
   - **Страница «Экспорт» (`9_Экспорт.py`)**:
     - Кнопки для скачивания портфеля, транзакций, алертов, отчётов в CSV.
   - **Страница «Мониторинг» (для администратора)** – если уже есть, дополнить данными из `/admin/health/all`, `/admin/metrics`, `/admin/system-info`. Если нет, создать `pages/10_Мониторинг.py`.

3. **Обновить существующие страницы**:
   - **Портфель** – добавить кнопку экспорта CSV, отображение AI-сигналов для монет портфеля.
   - **Уведомления** – добавить возможность создавать/удалять алерты (уже есть, но проверить совместимость с новым API).
   - **Админ-панель** – добавить вкладки: пользователи (список, управление подпиской), системные логи, графики метрик.

4. **Обновить боковую панель**:
   - Добавить ссылки на новые страницы.
   - Для администратора показывать страницы «Мониторинг» и «Админ-панель».

5. **Добавить в `requirements.txt`** (если чего-то не хватает):
   - `pandas`, `plotly` уже есть. Добавить `requests` (если нет). `aiohttp` не требуется, если используем `requests`.

6. **Проверить, что все страницы работают с реальным API**:
   - Подключиться к API_BASE_URL (он должен быть определён в `.env` или Secrets Replit).
   - Для теста можно использовать заглушки, если API на VPS ещё не развёрнут, но в коде должны быть реальные вызовы.

7. **Выполнить тестирование**:
   - Запустить дашборд (Streamlit) и убедиться, что нет ошибок импорта.
   - Открыть каждую новую страницу, проверить, что данные загружаются (хотя бы сообщение об ошибке, если API не отвечает).

### Что не нужно делать
- Не менять существующую структуру проекта (корневые файлы `app.py`, `.streamlit/config.toml` и т.д. – трогать только по необходимости).

- Не создавать отдельный бэкенд – только клиентские вызовы к API.
